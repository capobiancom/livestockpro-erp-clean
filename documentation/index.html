LivestockPro ERP Documentation

Complete guide for installing, configuring, and using LivestockPro ERP — a comprehensive farm and livestock management system.

Laravel 11 Vue 3 + Inertia.js MySQL / SQLite PHP 8.2+

Overview

LivestockPro ERP is a feature-rich, self-hosted farm and livestock management system built with Laravel 11 and Vue 3. It covers every aspect of running a modern farm operation — from animal health tracking and breeding management to payroll, accounting, and production reports.

🐄 Livestock Management

Track animals, herds, breeds, pregnancies, calvings, and reproduction records.

🏥 Health & Vaccination

Log health events, treatments, disease records, and vaccination schedules.

🌾 Feeding & Inventory

Manage feed types, feeding records, medicines, and stock movements.

🥛 Milk Production

Record daily milk yields per animal and manage milk sales with invoicing.

👥 HR & Payroll

Employee management, attendance, leave requests, and monthly payroll runs.

💰 Accounting

Full double-entry accounting: chart of accounts, journal entries, balance sheet, P&L, cash flow.

📊 Reports

15+ specialized reports: fertility, vaccination, inventory, financial, and more.

🏢 Multi-Farm SaaS

Optionally run as a multi-tenant SaaS with subscription plans and payment gateways.

Installation Guide

LivestockPro ERP includes a web-based Installation Wizard. You do not need SSH or command-line access to install it.

Method 1: Web Installer (Recommended)

1

Upload Files

Upload all files to your web server's public_html folder (or a subdirectory). Ensure the public/ folder is the document root.

2

Create Database

Create a MySQL database in your cPanel or hosting panel. Note down the database name, username, and password.

3

Open the Installer

Visit https://yourdomain.com/install in your browser. The Installation Wizard will guide you through the rest.

4

Follow the Wizard

Complete all 7 steps: Welcome → Server Check → Database → App Settings → Migrations → Admin Account → Done.

5

Log In

After installation, visit /login with your admin credentials.

Method 2: Command Line

If you have SSH/terminal access:

# 1. Install PHP dependencies
composer install --no-dev --optimize-autoloader

# 2. Copy and configure environment
cp .env.example .env
# Edit .env with your DB credentials and APP_URL

# 3. Generate application key
php artisan key:generate

# 4. Run migrations and seed initial data
php artisan migrate --force
php artisan db:seed --force

# 5. Create storage link
php artisan storage:link

# 6. Optimize
php artisan optimize

# 7. Create admin user (replace with your details)
php artisan tinker
# In tinker:
# $user = \App\Models\User::create(['name'=>'Admin','email'=>'admin@example.com','password'=>bcrypt('password')]);
# $user->assignRole('Super Admin');

Apache / .htaccess Configuration

Ensure mod_rewrite is enabled and AllowOverride All is set in your virtual host config. The included public/.htaccess handles URL rewriting automatically.

Nginx Configuration

server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/livestockpro/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }
    location ~ /\.(?!well-known).* { deny all; }
}

SaaS vs Single License Mode

LivestockPro ERP supports two operational modes, controlled by the SAAS_MODE environment variable in .env:

Setting Behavior Best For
SAAS_MODE=false All features unlocked. No subscription billing. All farms get full access. Single farm. Self-hosted personal use. Regular CodeCanyon buyer.
SAAS_MODE=true Subscription gates enforced. Farms must have an active paid plan to access features. Running as a SaaS business for multiple paying customers.
You can change SAAS_MODE at any time in your .env file, then run php artisan optimize:clear to apply the change.

In SaaS mode, the admin can:

  • Create subscription plans with monthly/yearly pricing
  • Assign features (feature flags) to each plan
  • Monitor farm subscriptions and payments
  • Inactivate subscriptions for non-paying farms

Payment Gateway Setup

LivestockPro ERP supports Stripe and bKash for subscription payments (only relevant when SAAS_MODE=true).

Stripe

1

Get API Keys

Log in to dashboard.stripe.com → Developers → API Keys.

2

Add to .env

Set STRIPE_KEY, STRIPE_SECRET, and STRIPE_WEBHOOK_SECRET in your .env file.

3

Enable in Admin

Go to Admin → Settings → Payment Gateways and enable Stripe.

4

Webhook

Add webhook URL https://yourdomain.com/payments/webhook/stripe in Stripe dashboard for payment confirmation events.

bKash (Bangladesh)

Add your bKash tokenized API credentials to .env: 

BKASH_APP_KEY=your_app_key
BKASH_APP_SECRET=your_app_secret
BKASH_USERNAME=your_username
BKASH_PASSWORD=your_password
BKASH_BASE_URL=https://tokenized.pay.bka.sh/v1.2.0-beta  # Production URL

Then enable bKash in Admin → Settings → Payment Gateways.

Email Setup

Configure outbound email for password resets, notifications, and demo request emails. Update these values in .env: 

MAIL_MAILER=smtp
MAIL_HOST=smtp.gmail.com          # Your SMTP host
MAIL_PORT=587
MAIL_USERNAME=your@email.com
MAIL_PASSWORD=your_app_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=your@email.com
MAIL_FROM_NAME="LivestockPro ERP"

Alternatively, configure email through the Admin panel at Admin → Settings → Email without editing .env directly.

For Gmail, use an App Password (not your regular password). Enable 2FA and create an App Password at myaccount.google.com/apppasswords.

File Permissions & Ownership

Proper file permissions and ownership are crucial for LivestockPro ERP to function correctly. The web server process (usually www-data or nginx) needs write access to certain directories.

Good news: The installation wizard automatically applies the correct permissions and ownership during setup. You only need to use these manual steps if permissions are misconfigured or after moving files to a production server.

Permission Requirements

Directory/File Permission Why
storage/ 755 Web server needs to write logs, uploads, and cache files
bootstrap/cache/ 755 Web server needs to cache compiled service providers
Other directories 755 Readable and executable (secure default)
Security Note: 755 permissions allow all users to read/write. This is acceptable for storage directories on dedicated servers, but on shared hosting you should use 775 with proper group ownership to the web server group. Never use 755 for files containing secrets.

Method 1: Automatic (Web Installer)

During the installation wizard, the system automatically:

  • Creates required directories if missing
  • Sets correct permissions using chmod
  • Attempts to change ownership to www-data (if running as root/sudo)

Simply access http://yourdomain.com/install and complete all 7 steps. Permissions are applied automatically.

Method 2: Manual - Using Artisan Command (SSH)

For existing installations or if automatic permissions failed, use the built-in artisan command: 

# Check if permissions need fixing (non-destructive)
php artisan app:fix-permissions

# Apply with sudo (requires root)
sudo php artisan app:fix-permissions

# For Nginx (different user)
sudo php artisan app:fix-permissions --user=nginx

Output example:

✓ chmod app              755
✓ chmod bootstrap        755
✓ chmod storage          755
✓ chmod bootstrap/cache  755
✓ chown storage          www-data:www-data
✓ chown bootstrap/cache  www-data:www-data
...
Successfully applied permissions!

Method 3: Manual - Direct SSH Commands

If you prefer direct shell commands, adjust the path and user as needed: 

# Navigate to your install directory
cd /var/www/livestockpro

# Set ownership to www-data
sudo chown -R www-data:www-data .

# Set default directory permissions (755 = read+execute for all, write for owner)
sudo chmod -R 755 .

# Set writable directory permissions (755 = read+execute for all, write for owner)
sudo chmod -R 755 storage
sudo chmod -R 755 bootstrap/cache

# Optional: Set .env to read-only for security
sudo chmod 644 .env

Method 4: cPanel File Manager

If you're on shared hosting with cPanel but no SSH access:

1

Login to cPanel

Access your cPanel dashboard (usually yourdomain.com:2083 or via hosting provider's panel).

2

Open File Manager

Navigate to Files → File Manager and ensure "Show Hidden Files" is enabled.

3

Select storage/ Directory

Right-click storage → Change Permissions → Set to 755 or 755. Enable "Recurse into subdirectories".

4

Repeat for bootstrap/cache

Right-click bootstrap/cache → Change Permissions → Set to 755 or 755. Enable "Recurse into subdirectories".

5

Verify

Run the installer wizard at http://yourdomain.com/install → Step 2 (Requirements). All writable directories should show ✓.

Troubleshooting

Issue: "The directory storage is not writable."

  • SSH: Run sudo chmod -R 755 storage bootstrap/cache
  • cPanel: Right-click → Change Permissions → 755 → Recurse
  • Check: ls -la storage/ — Look for rwx permissions for group/other

Issue: "500 error after upload to server."

  • SSH: sudo chown -R www-data:www-data . && sudo chmod -R 755 . && sudo chmod -R 755 storage bootstrap/cache
  • Ensure .env file exists and is readable
  • Check PHP error logs: tail -f /var/log/apache2/error_log

Issue: "Permission denied" when running artisan commands.

  • Ensure you're in the correct directory: cd /path/to/livestockpro/
  • If using SSH as non-root user and ownership is www-data: sudo php artisan app:fix-permissions
  • Or run as www-data: sudo -u www-data php artisan optimize:clear

Testing Permissions

Verify permissions are correct using these commands:

# Check directory permissions
ls -la storage/
ls -la bootstrap/cache/

# Create test file to verify writability
touch storage/test.txt
echo "test" > bootstrap/cache/test.txt

# Remove test files
rm storage/test.txt bootstrap/cache/test.txt

# Run installer requirements check

Frequently Asked Questions

Can I use this on shared hosting (cPanel)?

Yes! The installer works via browser. You need PHP 8.2+ and MySQL. Most shared hosting plans support this.

Does the installer need composer/npm to run?

No. The release zip includes pre-built vendor/ PHP dependencies and compiled public/build/ assets. You only need a web browser.

How do I reset the installation wizard?

Delete the file storage/installed to unlock the wizard. Note: this does not reset the database.

I get "Writable: storage — Not Writable". What do I do?

SSH: chmod -R 775 storage bootstrap/cache. On cPanel, use File Manager to change permissions to 755 or 775.

Can I run multiple farms from one installation?

Yes. In SaaS mode (SAAS_MODE=true), each farm is isolated by farm_id. Farms are created via the registration flow.

How do I change the default currency?

Log in as Super Admin → Admin panel → Settings → Website → set Currency. Per-farm currency can also be set during farm setup.

Can I disable the SaaS billing/subscription feature?

Yes. Set SAAS_MODE=false in .env and run php artisan optimize:clear. All features become freely accessible.

How do I add an SMTP email provider?

Update the MAIL_* variables in .env or use Admin → Settings → Email in the admin panel.

The application shows a 500 error. How do I debug?

Temporarily set APP_DEBUG=true in .env to see detailed errors. Remember to set it back to false afterward.
