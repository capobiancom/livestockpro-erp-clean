About This Guide

This document describes every menu link and the primary workflows (processes) in LivestockPro ERP. It mirrors the application sidebar (see resources/js/Pages/Layout/AppLayout.vue).

Each section includes: what the screen is for, common tasks, and a recommended workflow. Where applicable, it also lists the related URL path and typical records involved.
SaaS vs Single License: Some menus appear only in SaaS mode or only for Super Admin/Admin. Farm Owner menus are hidden in single-license mode.

Dashboard

Path: /dashboard The farm owner dashboard provides an at-a-glance summary of KPIs such as herd size, milk production, upcoming vaccinations, recent sales, and cash balances.

Screenshot: Dashboard overview (add file: documentation/screenshots/dashboard.png)

Common Tasks

  • Review alerts and upcoming tasks (vaccinations, checkups, low stock)
  • Use quick actions to add animals, record milk, or create invoices
  • Open linked reports from widgets to investigate anomalies

Workflow: Daily Check

1

Review alerts

Open Dashboard and check any warning/overdue items first.

2

Confirm today’s operations

Check vaccinations due, pregnancy checkups, and feeding/milk entries.

3

Drill down

Click widgets to open detailed lists/reports when something looks unusual.

Animals

Path: /animals Animal profiles are the core of the system. Each animal stores identifiers (tag), breed, herd, status, and links to reproduction, health, feeding, and production records.

Screenshot: Animals list (add file: documentation/screenshots/animals.png)

Common Tasks

  • Create animals and maintain accurate tag IDs
  • Update status (active/sold/dead) and herd assignment
  • Open an animal to review timeline: health, reproduction, milk, feeding

Workflow: Add New Animal

1

Open create form

Menu → Animals → Animals → Add new record.

2

Fill identity & grouping

Enter Tag ID, name, breed, sex, DOB, herd, and status.

3

Save and verify

Save and confirm the animal appears in the list and dashboard counts.

Tips

  • Use consistent tag formats (example: FARM-0001)
  • Attach photos/documents for traceability

Installation

High-level installation steps (web installer and CLI). For the full installer walkthrough, see documentation/index.html.

Web installer URL: /install

Prerequisites

  • PHP 8.2+ and required extensions
  • MySQL/SQLite configured
  • Writable: storage/, bootstrap/cache/

Backup & Restore

Best practices for backups: export database and archive storage/ and public/ uploads. Keep copies off-site and test restores periodically.

Commands

# MySQL dump example
mysqldump -u dbuser -p livestockpro > livestockpro.sql

# Tar backup of uploads and storage
tar -czf backup-storage-$(date +%F).tar.gz storage public/uploads

Updates

Updating the application involves replacing code files (or pulling new release), running composer, migrations, and clearing caches.

Typical Update Steps

  1. Backup DB and storage
  2. Upload new files (do not overwrite .env or storage)
  3. SSH: composer install --no-dev --optimize-autoloader
  4. SSH: php artisan migrate --force && php artisan optimize:clear

Troubleshooting

Common issues and how to resolve them:

Installer Failures

  • Missing .env: index.html installer creates a default; ensure web server can write to webroot
  • Permission errors: run sudo php artisan app:fix-permissions or adjust with chown/chmod as documented in index.html

500 Errors

  • Enable APP_DEBUG=true in .env (development only) and check storage/logs/laravel.log
  • Check PHP error logs on the server (Apache/Nginx)

Performance

  • Use PHP-FPM with tuned worker settings, enable opcache
  • Use Redis for cache/queue in high-traffic setups

Appendix — Helpful Commands & Notes

  • Clear caches: php artisan optimize:clear && php artisan optimize
  • Queue worker: php artisan queue:work --sleep=3 --tries=3
  • Scheduler: add cron * * * * * php /path/to/artisan schedule:run
  • Fix permissions: sudo php artisan app:fix-permissions
Screenshot support is built-in: add PNG/JPG files into documentation/screenshots/ using the filenames referenced in each section (example: dashboard.png). Missing images are automatically hidden.
